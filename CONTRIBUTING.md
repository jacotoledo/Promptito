# Contributing to Promptito

Thank you for your interest in contributing to Promptito!

## How to Contribute

### Bug Reports

- Search existing issues first
- Use the bug report template
- Include Go version, OS, and steps to reproduce

### Feature Requests

- Describe the problem you're solving
- Explain why it fits the project goals
- Keep scope focused

### Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Make your changes
4. Run tests: `go test ./...`
5. Commit with clear messages
6. Push and create a Pull Request

## Code Standards

- Follow Go conventions (run `go fmt`)
- Add tests for new features
- Update documentation
- Keep commits atomic

## SKILL.md Contributions

When adding sample prompts:

- Follow the [SKILL.md schema](./SKILL.md)
- Include realistic examples
- Use semantic versioning
- Add appropriate tags and categories

## Security

For security issues, see [SECURITY.md](./SECURITY.md).

**Do NOT** open public issues for security vulnerabilities.
